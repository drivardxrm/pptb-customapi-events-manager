## Implementation Pattern

**Cross-Component State Handoff:**
```
CatalogModal (create) 
  → stores created ID via useAppStore
  → closes modal

BusinessEventDetails (listening)
  → detects pending catalog ID
  → triggers data refresh
  → selects new catalog in tree/details
  → clears pending ID
```

**Pending State Guard:**
- Pending ID only used once per component mount cycle
- Cleared after selection completes to prevent stale re-activation
- Matches existing assignment handoff lifecycle
