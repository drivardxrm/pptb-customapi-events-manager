## Context

Recent `CatalogModal.tsx` changes moved the publisher picker UI over to Zustand-backed `selectedPublisherId`, but the create form stopped carrying its own publisher value. This created two regressions:
1. The default publisher from app settings no longer rehydrated reliably when the modal opened before settings finished loading
2. The submitted create payload could diverge from the publisher shown in the UI

## Decision & Implementation Path

Treat the catalog create modal's publisher as **form-owned payload state first** and mirror it to Zustand only as a convenience for later create flows:
- Initialize the publisher from `defaultPublisherId` once per modal open
- Stop auto-applying settings after first hydration so manual changes remain respected
- Translate selected publisher to `PublisherId@odata.bind` at submit time

**Key Files:**
- `src/components/BusinessEventDetails/CatalogModal.tsx`
- `src/hooks/useAppSettings.ts`
- `src/models/AppSettings.ts`
