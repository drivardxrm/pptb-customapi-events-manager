## Root Cause Analysis

### Problem Statement
Saving a newly created catalog triggered a React error 310 (too many renders / hook mismatch) in production.

### Investigation
Analyzed `CatalogTreeView` component for conditional hook usage patterns. Found the issue:
- Component execution flow had conditional early returns before `useMemo` call
- When tree selected a newly created catalog, the component path changed
- Different code paths executed different numbers of hooks before returns
- React detected hook count mismatch between renders

### Root Cause
`useMemo` was conditionally computed—only executed on certain code paths. React requires hooks to always execute in the same order and count.

## Solution Implemented

**Component:** `src/components/BusinessEventDetails/CatalogTreeView.tsx`

**Change:** Move `allOpenItems` calculation before any early returns to ensure unconditional execution.

```typescript
// BEFORE (broken):
if (!catalogs || catalogs.length === 0) return null;

const allOpenItems = useMemo(() => {
  // calculation here
}, [catalogs, selectedCatalogId, selectedTreeItemId]);

// AFTER (fixed):
const allOpenItems = useMemo(() => {
  // calculation here
}, [catalogs, selectedCatalogId, selectedTreeItemId]);

if (!catalogs || catalogs.length === 0) return null;
```
