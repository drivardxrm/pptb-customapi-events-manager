## Work Completed

Implemented a shared `OpenBusinessEventAction` utility for Custom API to Business Event navigation. The feature enables users to jump from Custom API Details or Tester to Business Event view with the corresponding catalog assignment selected.

### Implementation Pattern

- **Handoff Mechanism:** Uses Zustand `pendingBusinessEventAssignmentId` to carry selected catalog assignment across nav changes
- **Single Assignment:** Direct navigation to Business Event without prompting
- **Multiple Assignments:** Chooser dialog displays all assigned catalogs; user selects one to navigate
- **State Management:** Clears pending ID when leaving Business Event view to prevent abandoned handoff replay
- **Tree Selection:** Auto-derives root catalog from assignment category; expands tree to show selection

### Technical Approach

1. Query catalog assignments for the current Custom API (filtered by `_object_value@Microsoft.Dynamics.CRM.lookuplogicalname === 'customapi'`)
2. Evaluate assignment count:
   - 0: No button appears
   - 1: Set `pendingBusinessEventAssignmentId` + navigate to Business Event
   - 2+: Show chooser dialog with catalog options
3. In `BusinessEventDetails`, wait for catalogs + assignments to load, then reconstruct tree selection from pending ID
4. Clear pending ID on unmount or nav away
