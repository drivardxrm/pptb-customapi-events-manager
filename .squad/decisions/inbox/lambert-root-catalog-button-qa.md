## Scope

This QA checklist covers the feature to add a "New Root Catalog Button" in the Business Events Details form, reusing the existing `CatalogModal` component while:
- Matching the visual style and behavior of the "New Custom API" button
- Setting the modal to "Create Root Catalog" mode (not "Create Category")
- Ensuring no parent catalog is displayed for root creation
- Preserving existing create-category and edit behaviors

---

## Button Visibility & Styling

### ✅ Button Display
- [ ] **Visibility Requirement:** "New Root Catalog" button appears in the Business Events header action area
- [ ] **Solution-Scoped Display:** Button is **only visible when a solution is selected** (`selectedSolutionId` is truthy)
- [ ] **No Solution Selected:** Button is **hidden** when no solution is selected
- [ ] **Correct Location:** Button appears next to (or in) the header action group of the Business Events card

### ✅ Style Parity with "New Custom API" Button
- [ ] **Button Appearance:** Uses `appearance="secondary"` (matching Custom API button)
- [ ] **Button Icon:** Uses `AddCircleColor` icon (matching Custom API button)
- [ ] **Button Text:** Displays "New Root Catalog" text
- [ ] **CSS Classes:** Applied with `className={styles.headerActionButton}` (consistent styling)
- [ ] **Hover/Focus States:** Button respects Fluent UI theme hover and focus states (no custom overrides)

## Modal Behavior: Create Root Catalog Mode

### ✅ Modal Title & Labels
- [ ] **Modal Title:** Displays "Create Root Catalog" (not "Create Category")
- [ ] **Title Source:** Title is generated by `CatalogModal.getTitle()` returning correct value for `mode === 'create-root'`

### ✅ Parent Catalog Display
- [ ] **No Parent Display:** Parent catalog info block is **NOT shown** when creating root catalog
- [ ] **Conditional Rendering:** Parent info only displays when `isCategory && parentCatalog` both true
- [ ] **Root Case:** When `mode === 'create-root'`, `parentCatalog` is null, so parent info does not render

### ✅ Form Fields for Root Creation
- [ ] **Publisher Field:** Visible and required for root catalog creation (not for category creation)
- [ ] **Unique Name Field:** Visible and required for root catalog creation
- [ ] **Name Field:** Visible and required
- [ ] **Display Name Field:** Visible and required
- [ ] **Description Field:** Visible and optional
- [ ] **Validation:** All required fields validate before Save is enabled

### ✅ Modal Actions
- [ ] **Cancel Button:** Closes modal without saving
- [ ] **Save Button:** Enabled only when validation passes
- [ ] **Save Button (Disabled State):** Disabled while form is invalid or saving
- [ ] **Spinner During Save:** Spinner icon appears in Save button while request is in flight
- [ ] **Success Close:** Modal closes after successful creation

2. [ ] User clicks "New Root Catalog" button in Business Events header

## Error & Edge Cases

### ✅ Validation Errors
- [ ] **Missing Publisher:** Error message appears when Publisher not selected for root creation
- [ ] **Missing Name:** Error message appears when Name is empty
- [ ] **Missing Display Name:** Error message appears when Display Name is empty
- [ ] **Missing Unique Name:** Error message appears when Unique Name is empty for root creation
- [ ] **Duplicate Unique Name:** Error message if Unique Name already exists

### ✅ API Error Handling
- [ ] **Network Error:** Modal remains open; error message displayed; Save button re-enabled
- [ ] **Dataverse Error:** Error message shown; modal can be closed or form re-attempted
- [ ] **Spinner Behavior:** Spinner displays during request and is removed on success/failure

### ✅ Edge Cases
- [ ] **No Solution Selected:** "New Root Catalog" button is hidden; tree view shows "No Catalog selected"
- [ ] **Solution Change:** If solution is changed while modal is open, modal closes gracefully
- [ ] **Rapid Clicks:** Multiple rapid clicks of "New Root Catalog" button do not open multiple modals
